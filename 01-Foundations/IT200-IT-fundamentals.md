  scenario-based organizational problems

## What I understand now that I didn't before
- Systems thinking changes how you approach problems.
  Instead of looking at individual components in isolation
  you look at how they interact, where feedback loops
  exist, and how a change in one part affects the whole
  system. This is directly applicable to understanding
  how malware affects an entire environment rather than
  just one file or process.
- Hardware knowledge matters for security. Understanding
  how components interact at a physical level — memory,
  storage, CPU — builds the foundation for understanding
  how malware manipulates those same components at a
  software level.
- Ethics in IT is not an abstract concept. Identifying
  phishing risks in this course was my first exposure to
  thinking about how human behavior is exploited as an
  attack vector — a theme that runs through every area
  of cybersecurity.

## Connection to malware analysis
- **Systems thinking is how malware analysts approach
  an infected environment.** Malware does not exist in
  isolation — it affects processes, registry, network,
  and files simultaneously. Thinking in systems rather
  than individual components is essential for understanding
  the full impact of an infection.
- **Hardware fundamentals underpin memory forensics.**
  Understanding how RAM, storage, and CPU work at a basic
  level makes advanced topics like memory forensics and
  process injection more accessible because the physical
  foundation is already in place.
- **Phishing awareness connects to malware delivery.**
  Most malware enters an environment through phishing.
  IT 200 introduced the human factor in security —
  understanding how malware is delivered is as important
  as understanding what it does after delivery.
